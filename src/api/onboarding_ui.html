Getting Started
1️⃣ Create Your Workspace

Set up your workspace and get your API key. You'll need the admin key from your deployment.

curl -X POST http://localhost:8000/api/v1/workspaces \
  -H "Authorization: Bearer YOUR_ADMIN_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "My Company", "plan": "pro"}'
2️⃣ Configure ICP Criteria

Define your Ideal Customer Profile — industries, company size, revenue range, and scoring weights.

curl http://localhost:8000/admin/config/icp \
  -H "Authorization: Bearer cmo_YOUR_API_KEY"
3️⃣ Connect Your CRM

Add your HubSpot or Zoho API keys in .env. Seller briefs and scores will be written back automatically.

# Add to your .env file:
CMO_HUBSPOT_API_KEY=your_hubspot_key
# or
CMO_ZOHO_CLIENT_ID=your_zoho_id
CMO_ZOHO_CLIENT_SECRET=your_zoho_secret
4️⃣ Upload Knowledge Base

Upload your battlecards, case studies, and messaging for personalized outbound.

curl -X POST http://localhost:8000/api/v1/kb/upload \
  -H "Authorization: Bearer cmo_YOUR_API_KEY" \
  -F "title=Competitor Battlecard" \
  -F "kb_type=battlecard" \
  -F "content=## Competitor X\n\nWeaknesses: ..."
5️⃣ Run Your First Qualification

Create a campaign and trigger account qualification to generate seller briefs.

# Create a campaign
curl -X POST http://localhost:8000/campaigns \
  -H "Authorization: Bearer cmo_YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name": "Q1 Outbound", "icp_criteria": {"industries": ["SaaS"]}}'

# Trigger qualification
curl -X POST http://localhost:8000/api/v1/campaigns/CAMPAIGN_ID/qualify \
  -H "Authorization: Bearer cmo_YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"max_accounts": 10}'
6️⃣ Review Seller Briefs

Check your generated briefs, export data, and monitor usage.

# Export briefs
curl http://localhost:8000/api/v1/export/briefs?format=json \
  -H "Authorization: Bearer cmo_YOUR_API_KEY"

# Check usage
curl http://localhost:8000/api/v1/usage \
  -H "Authorization: Bearer cmo_YOUR_API_KEY"
